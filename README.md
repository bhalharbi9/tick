# UCL Predictions — GitHub Pages + Supabase

نسخة ثابتة تعمل على **GitHub Pages** مع **Supabase** بدون PHP أو MySQL.

## الميزات
- التوقع لمباريات **الإياب فقط**
- عرض **نتيجة الذهاب** للتذكير
- إغلاق التوقع تلقائيًا قبل المباراة بعدد دقائق قابل للتعديل
- دخول المشارك **بالاسم فقط**
- جدول ترتيب مباشر مع **Auto Refresh**
- كروت **Top 5**
- جوائز تلقائية:
  - الأول: 30 ريال
  - الثاني: 20 ريال
  - الثالث: 10 ريال
- **QR Code** للدخول السريع
- لوحة مشرف Bootstrap 5
- تصدير CSV
- حفظ التشكيلة وآخر 5 مباريات يدويًا من لوحة المشرف

## مهم
هذه النسخة **لا تجلب النتائج المباشرة تلقائيًا من 365Scores**.  
السبب أن GitHub Pages مشروع ثابت، والربط المباشر مع مواقع خارجية بهذا الشكل غير مستقر.  
النتائج والتشكيلات وآخر 5 مباريات تُدار من **لوحة المشرف**.

## الملفات
- `index.html` صفحة التوقعات
- `leaderboard.html` صفحة الترتيب المباشر
- `admin.html` لوحة المشرف
- `schema.sql` جداول Supabase + السياسات + View
- `assets/js/config.example.js` مثال إعداد الاتصال
- `assets/js/*.js` ملفات الجافاسكربت
- `assets/css/style.css` التنسيقات

## الخطوات

### 1) إنشاء مشروع Supabase
أنشئ مشروعًا جديدًا، ثم افتح **SQL Editor** ونفّذ الملف `schema.sql`.

### 2) إنشاء حساب مشرف
من Supabase:
- Authentication
- Users
- Add user

استخدم هذا الحساب للدخول إلى `admin.html`.

### 3) إعداد الاتصال
انسخ الملف:
`assets/js/config.example.js`
إلى:
`assets/js/config.js`

ثم ضع:
- `SUPABASE_URL`
- `SUPABASE_ANON_KEY`

### 4) رفع المشروع
ارفع الملفات إلى GitHub Pages.

### 5) إضافة المباريات
من `admin.html`:
- أضف مباريات الإياب
- ضع نتيجة الذهاب
- ضع وقت المباراة
- أدخل النتيجة الفعلية لاحقًا
- عند الحفظ يتم إعادة احتساب النقاط تلقائيًا

## ملاحظات الأمان
- المشاركون يدخلون بالاسم فقط، لذلك النسخة مناسبة لمسابقات بسيطة.
- المشرف فقط يستطيع تعديل المباريات والنتائج لأن ذلك محمي بحساب Supabase Auth.
- إذا أردت لاحقًا حماية أكبر للمشاركين، يمكن إضافة OTP أو رابط شخصي لكل مشارك.

## النشر على GitHub Pages
يمكنك استخدام أي اسم مستودع.  
إذا كان الرابط النهائي مثل:
`https://username.github.io/ucl-predictions/`
فسيظهر QR داخل الواجهة تلقائيًا بناءً على رابط الصفحة.

## الترتيب
يعتمد على:
1. مجموع النقاط
2. عدد النتائج الدقيقة
3. عدد التوقعات القريبة
4. أقل مجموع مسافة خطأ

## منطق النقاط
- 10 نقاط: النتيجة الدقيقة
- 7 نقاط: فارق الأهداف صحيح + اتجاه المباراة صحيح
- 5 نقاط: اتجاه المباراة صحيح
- +3 نقاط: إذا أصاب أهداف أحد الفريقين بالضبط
- +2: إذا كانت مسافة الخطأ = 1
- +1: إذا كانت مسافة الخطأ = 2
- الحد الأعلى لغير النتيجة الدقيقة = 9

## تصدير PDF
يوجد زر طباعة مناسب للحفظ كـ PDF من المتصفح مباشرة.
